# Product

## Register

product

## Users

Developers and technical power users who spend hours in an AI coding interface. They manage long-running agent sessions, switch between models mid-conversation, review tool outputs, and expect a professional developer-tool experience comparable to Claude Code, Cursor, or Vercel. They work in the terminal primarily but use the web UI for session review, sharing, and cloud features.

## Product Purpose

Squido's web interface is the browser-based frontend for interacting with AI coding agents. It provides session management, tool visibility, agent status awareness, and a polished chat experience for developers who want the power of a CLI agent in a GUI environment. The Agent Chat page is the primary surface — the place where the user spends nearly all their time.

## Brand Personality

Confident, professional, and technical. Warm amber accent on a deep ocean dark background. Developer-tool aesthetic that earns trust through clarity, density when needed, and precision. Not playful or overly branded — the medium is the message.

## Anti-references

- Basic chat UIs that look like generic messaging apps
- SaaS-cream/beige backgrounds with generic illustrations
- Over-decorated interfaces that prioritize visual novelty over usability
- Interfaces where it's not immediately clear what the agent is doing (status opacity)
- Flat gray corporate dashboards with no personality

## Design Principles

1. **The tool disappears into the task** — every decorative choice must earn its place against the user's flow state. Unnecessary chrome is noise.
2. **Ground in the subject** — terminal, code, and developer tools are the natural world of this product. Draw from that vernacular: monospace where appropriate, terminal motifs, code-like precision.
3. **Structure is information** — layout, spacing, and hierarchy encode meaning. The three-panel architecture (sessions / chat / context) exposes state without hiding it behind clicks.
4. **Earned familiarity** — the bar is a user fluent in Linear, Raycast, and Cursor sitting down and trusting the interface immediately. Standard patterns (top bar + side nav, command palette, resizable panels) are virtues, not cop-outs.
5. **One bold choice, discipline everywhere else** — spend the memorable element in one place (the terminal demo / agent response rhythm); keep everything around it quiet and precise.

## Accessibility & Inclusion

WCAG AA minimum. Keyboard-navigable throughout. Respect prefers-reduced-motion. Dark-mode first (existing). Maintain sufficient color contrast on all surfaces — the amber brand on dark backgrounds is deliberately warm but must pass AA thresholds.
